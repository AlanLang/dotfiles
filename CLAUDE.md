# CLAUDE.md

This is a personal dotfiles repository managed with [GNU Stow](https://www.gnu.org/software/stow/), plus a macOS bootstrap script (`install.sh` + `scripts/`) that sets up a fresh machine end-to-end (merged from the former `macos-start` repo).

## Repository Structure

Each top-level config directory is a Stow package. Running `stow <dir>` from the repo root creates symlinks in `$HOME` mirroring the directory's structure.

```
dotfiles/
├── git/            # .gitconfig, .config/git/ignore
├── zsh/            # .zshrc (Oh My Zsh + Starship) + .config/starship.toml
├── nvim/           # .config/nvim/ (Lua config, lazy.nvim)
├── vim/            # .vimrc
├── tmux/           # .tmux.conf, .tmux.conf.local
├── ghostty/        # .config/ghostty/config
├── lazygit/        # .config/lazygit/config.yml
├── hammerspoon/    # .hammerspoon/init.lua (macOS only, 按 App 自动切换输入法)
├── rime/           # Library/Rime/*.custom.yaml 等（macOS only, 鼠须管自定义配置）
├── iterm2/         # iTerm2 profile JSON export
├── vscode/         # VS Code user settings (macOS path)
├── Brewfile        # CLI tools + fonts + GUI apps, installed by scripts/apps.sh
├── install.sh      # macOS one-shot bootstrap (runs scripts/ in order)
├── scripts/        # bootstrap steps: homebrew/apps/dotfiles/zsh/node/nvim/rime/macos
└── start.sh        # Quick server bootstrap (vim + tmux + fish), downloads from GitHub
```

## Key Conventions

- **Language**: Config comments and sub-READMEs are in Chinese (Simplified).
- **Shell**: Primary shell is Zsh with Oh My Zsh + Starship prompt.
- **Editor**: Neovim is the primary editor (`nvim/`). Vim config (`vim/`) is a lightweight fallback for servers.
- **Input method**: Rime 鼠须管 (squirrel) with rime-ice + 小鹤双拼; Hammerspoon switches input source per app.
- **Font**: FiraCode Nerd Font is expected (configured in Ghostty).
- **Theme**: Catppuccin Mocha is the preferred color palette (used in Starship config).
- **Package manager**: Bun is configured in zsh PATH.

## Working with This Repo

- Adding a new tool: create a Stow package mirroring `$HOME`, add it to `PACKAGES` in `scripts/dotfiles.sh`, add the cask to `Brewfile` if it's a GUI app, and update the README tables.
- `scripts/dotfiles.sh` stows from the repo itself (no clone); it pre-creates target dirs (`~/.hammerspoon`, `~/Library/Rime`, `~/.config`, VS Code `User/`) so Stow links individual files instead of whole directories — runtime files must not end up in the repo.
- The `zsh` package is stowed by `scripts/zsh.sh` (after Oh My Zsh install), not by `scripts/dotfiles.sh`.
- Do NOT modify `lazy-lock.json` manually — it is auto-generated by lazy.nvim.
- The `tmux/.tmux.conf` is a large vendored file (gpakosz/.tmux); local customizations go in `.tmux.conf.local`.
- `start.sh` is a quick bootstrap script for servers — it downloads configs directly from GitHub rather than using Stow.
- The `.gitignore` excludes `lazygit/.config/lazygit/state.yml` (runtime state).
